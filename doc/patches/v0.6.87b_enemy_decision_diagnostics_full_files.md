# v0.6.87b — Enemy decision diagnostics, complete files version

This is the no-patch-script version of v0.6.87.

## Installation

Copy the package contents over the root of the `EnemyTraceSimulator` repository.

No PowerShell script is needed.

## Files included

```text
scripts/tools/simulation/LadyBugEnemyDecisionModel.cs
scripts/tools/simulation/LadyBugStaticMazeRomTable.cs
scripts/tools/simulation/LadyBugEnemyDecisionTraceDiagnosticAdapter.cs
scripts/tools/simulation/LadyBugEnemySimulationAdapter.cs
```

`LadyBugEnemySimulationAdapter.cs` is a complete replacement file. It keeps the existing adapter registered under the same UI entry:

```text
Lady Bug reference-direction step
```

So the test stays exactly the same as before:

1. Rebuild in Godot / Rider.
2. Load your standard JSONL trace.
3. Open `Compare`.
4. Run `Lady Bug reference-direction step`.

Expected comparison result:

```text
mismatches=0
```

## Scope and limitation

This diagnostic starts validating the source-first skeleton around:

```text
0x427E  decision-center predicate
0x3911  logical maze validation
0x42E6  preferred direction decision path
0x4241  fallback scanner
```

It does not yet validate local-door/tile rejection faithfully, because the diagnostic still uses a permissive tile reader instead of a complete transcription of the `0x3C0A` tile lookup used by `0x4130`.

That means this is a safe diagnostic step, not the final independent enemy AI.
