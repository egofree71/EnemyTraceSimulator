# v0.6.88 — rejectedMask transition diagnostics

This package keeps the reference-synced simulation path unchanged.

It updates the source-first decision diagnostics to analyze transitions between
standard JSONL frames:

```text
previous frame EnemyWork temp state -> current frame EnemyWork result
```

The goal is to validate the `0x4315` case seen at tick 245:

```text
preferred candidate rejected
0x61C1 |= preferred
current temp direction remains valid
current direction is kept
```

This is the source-first interpretation of the observed shape:

```text
previous temp dir = 01
preferred[0]      = 08
current temp dir  = 01
MAME rejectedMask = 08
```

The older shadow heuristic classified that as `DECISION_CENTER_KEEP_CURRENT_NO_REJECT_WRITE`.
This package introduces the corrected diagnostic source:

```text
4315_PREFERRED_REJECTED_CURRENT_KEPT
```
