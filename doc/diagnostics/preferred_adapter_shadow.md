# preferred[] adapter shadow diagnostics

`v0.6.72` starts wiring `LadyBugMonsterPreferenceSystem` into the standard JSONL comparison path, but only in shadow/diagnostic mode.

The adapter still keeps:

```text
EnemyWork.Preferred
```

synchronized from MAME, so the existing comparison checkpoint should not be broken.

The new diagnostic path computes:

```text
EnemyWork.PreferredShadow
EnemyWork.PreferredShadowSource
```

from the C# model and classifies each end-of-frame preferred[] tuple from the standard JSONL trace.

Classification order:

1. exact rotate tuple from `2E97_ROTATE_FROM_08`;
2. exact random tuple from `2EC7_RANDOM_RLOW_<nibble>`;
3. slot-0 BFS/chase override shape:
   - base tuple generated by the C# model;
   - `preferred[0]` replaced by the observed value;
   - used only because exact-PC diagnostics proved current one-enemy captures override `61C4`.

The adapter result summary now includes:

```text
preferred[] shadow model checks=...
matches=...
mismatches=...
sources: ...
```

Important limitation:

This is not yet a full independent preferred[] implementation. The standard JSONL trace does not contain the exact write PC or the exact `LD A,R` value, so this adapter shadow step is a classification/compatibility check, not a cycle-accurate write replay.

The exact-PC `error.log` analyzer remains the source of truth for write-PC attribution.
