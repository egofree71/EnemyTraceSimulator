# Adapter preferred[] shadow mismatch details

`v0.6.74` improves the adapter-level preferred[] shadow diagnostics.

The previous adapter summary already reported:

```text
preferred[] shadow model checks=...
matches=...
mismatches=...
sources: ...
```

The new version keeps the same successful-path output, but if a mismatch appears in a future trace, the first mismatch now includes more context:

```text
tick=...
mameFrame=...
pc=...
r=...
activeEnemies=...
tempDir=...
tempX=...
tempY=...
chaseTimers=[...]
chaseRoundRobin=...
source=...
reference=[..]
shadow=[..]
```

This patch does not change simulation behavior:

- authoritative `EnemyWork.preferred[]` is still reference-synced from MAME;
- `PreferredShadow` is still diagnostic only;
- BFS/chase direction is still inferred from observed reference data when a slot-0 overlay is classified.

Purpose:

- make the next one-enemy traces easier to debug;
- avoid being blind if the shadow model fails on another save-state;
- prepare the future step where `preferred[]` can stop being reference-synced.
