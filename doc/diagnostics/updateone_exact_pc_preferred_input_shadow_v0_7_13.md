# v0.7.13 — Enemy_UpdateOne exact-PC preferred-input shadow

## Goal

v0.7.13 keeps the normal JSONL trace debugger-free and uses the separate exact-PC
EnemyWork diagnostic as an offline companion tape.

The new shadow verifies that `Enemy_UpdateOne` can consume `preferred[slot]` from
the imported exact-PC 0x2E5C tape window aligned in v0.7.12, instead of consuming
`preferred[slot]` from the standard-trace tuple classifier.

## Scope

Validated scope remains intentionally narrow:

- static player sequence
- one active enemy, slot 0
- standard trace: `ladybug_sequence_v8_fullmem_trace.jsonl`
- exact-PC companion: `tools/mame/lua/error.log` generated by
  `ladybug_enemywork_pc_trace.lua`

## What changed

New diagnostic summary:

```text
Lady Bug Enemy_UpdateOne exact-PC preferred-input shadow v0.7.13
```

It:

1. imports the exact-PC tape from `error.log`,
2. reconstructs complete 0x2E5C preferred[] calls,
3. finds the same contiguous tuple window as v0.7.12,
4. feeds the selected `preferred[slot]` from that exact-PC window,
5. compares it against the reference `EnemyWork.preferred[slot]` consumed by the
   current Enemy_UpdateOne shadow.

Expected result for the current reference sequence:

```text
exactProviderUsable=true
checks=496
matches=496
mismatches=0
bestWindowTupleMismatches=0
```

## Limits

This is still shadow-only.  It does not yet alter the visible simulation state and
does not prove exact per-frame PC timing, because the raw `error.log` LBEW lines
still do not contain the standard JSONL tick number.

The forced-reversal carry-set path remains outside the validated scope for this
sequence.
